# Target Files

- `overview.md`: 高层摘要、当前阶段、整体限制与重点资产
- `prd.md`: 需求背景、目标范围、验收口径
- `review-notes.md`: 评审结论、争议点、后续 action
- `frontend-design.md`: 前端页面范围、交互、状态、依赖
- `backend-design.md`: 后端接口、模型、兼容性、发布影响
- `test-cases.md`: 测试流程、边界场景、回归口径
- `development.md`: 当前需求点、实现备注、风险阻塞、Git 自动区之外的开发说明
- `decision-log.md`: 会被长期引用的结论、取舍、约束
- `commits.md`: 提交记录，不用于记普通需求补充
